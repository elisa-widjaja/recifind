# Story 08 — Capacitor Shell Bootstrap

> Part of [iOS App + Rebrand workstream](./2026-04-17-ios-app-workstream.md)
> REQUIRED SUB-SKILL: superpowers:subagent-driven-development

**Goal:** Stand up the `apps/ios/` workspace with Capacitor wrapping the existing React PWA. Run the app in iOS Simulator and on a real iPhone. Set URL scheme, Associated Domains, APNs entitlement, and PrivacyInfo.xcprivacy. Nothing but the shell — auth, share, push come in later stories.

**Depends on:** Story 01 (rebrand — bundle ID and app name exist), Story 02 (contracts — IOS constants)
**Blocks:** Stories 09, 10, 11 (all iOS features)
**Can develop in parallel with:** Stories 03, 04, 05, 06, 07

**Contracts consumed:** C5 iOS identifiers (BUNDLE_ID, URL_SCHEME, ASSOCIATED_DOMAIN, SHARE_EXT_BUNDLE_ID)
**Contracts produced:** Working `apps/ios/App.xcworkspace` for downstream stories to edit

## Task 2: Capacitor config

- [ ] **Step 1:** Create `apps/ios/capacitor.config.ts`

```typescript
import type { CapacitorConfig } from '@capacitor/cli';
import { IOS } from '../shared/contracts';

const config: CapacitorConfig = {
  appId: IOS.BUNDLE_ID,
  appName: IOS.APP_NAME,
  webDir: '../recipe-ui/dist',   // Capacitor copies this into the iOS bundle on `cap sync`
  ios: {
    scheme: IOS.URL_SCHEME,
  },
};

export default config;
```

## Task 4: Info.plist — URL scheme + permission strings

Open Xcode (`cd apps/ios && npx cap open ios`) and edit `App/App/Info.plist`.

- [ ] **Step 1:** Register the URL scheme

Add to `Info.plist`:

```xml
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLName</key>
    <string>com.recifriend.app</string>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>recifriend</string>
    </array>
  </dict>
</array>
```

- [ ] **Step 2:** Permission strings (required by iOS, even if we don't prompt yet)

```xml
<key>NSUserNotificationsUsageDescription</key>
<string>ReciFriend sends notifications when a friend shares a recipe with you or saves one of yours.</string>

<key>NSCameraUsageDescription</key>
<string>ReciFriend uses the camera to photograph your cooking so you can share it with friends.</string>

<key>NSPhotoLibraryUsageDescription</key>
<string>ReciFriend uses your photo library so you can attach recipe photos.</string>
```

- [ ] **Step 3:** Launch screen branding — update `LaunchScreen.storyboard` background color to match brand or leave default white. (Optional.)

- [ ] **Step 4:** Commit

```bash
git add apps/ios/App/App/Info.plist
```

## Task 5: Entitlements — Associated Domains + APNs

- [ ] **Step 1:** Create `apps/ios/App/App/App.entitlements`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>aps-environment</key>
  <string>development</string>

  <key>com.apple.developer.associated-domains</key>
  <array>
    <string>applinks:recifriend.com</string>
  </array>
</dict>
</plist>
```

- [ ] **Step 2:** In Xcode, add this file to the App target:
  - Select App target → Signing & Capabilities
  - `+ Capability` → **Push Notifications** (creates/references aps-environment)
  - `+ Capability` → **Associated Domains** → add `applinks:recifriend.com`

When App Store submission happens (Story 12), Xcode automatically flips `aps-environment` to `production`.

- [ ] **Step 3:** Commit

```bash
git add apps/ios/App/App/App.entitlements
git commit -m "feat(ios): add Associated Domains + push notification entitlement"
```

## Task 8: Document local dev

- [ ] **Step 1:** Create `docs/runbooks/ios-local-dev.md`

```markdown
# iOS local dev loop

1. Edit React code in `apps/recipe-ui/src/`
2. `cd apps/recipe-ui && npm run build`
3. `cd apps/ios && npx cap sync ios`
4. In Xcode: Cmd-R (simulator or device)

To test Universal Links: `https://recifriend.com/<path>` from Safari on device.
To test custom scheme: `recifriend://<path>` — in Safari, enter `recifriend://friend-requests`.

## Common gotchas
- **Web assets not updating**: did you `cap sync`? Capacitor caches aggressively.
- **White screen on launch**: check Xcode console for WebView errors.
- **Universal Link opens Safari instead of app**: AASA file broken. Check https://branch.io/resources/aasa-validator/
- **Crash on push registration**: APNs entitlement missing from App.entitlements
```
